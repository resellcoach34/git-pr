# Repository Harness Design

## 목적

이 저장소를 여는 모든 Codex 대화가 이전 대화 내용을 몰라도 현재 운영 상태, 작업 방향, 품질 기준, 배포 절차를 즉시 파악하고 능동적으로 다음 행동을 수행하게 한다.

하네스에서 “작업 완료”는 로컬 파일 수정이 아니라 다음 전체 과정의 완료를 뜻한다.

1. 요구사항 확인
2. 회귀 테스트 추가 또는 갱신
3. 구현
4. 자동 테스트와 시각 검증
5. Git 커밋과 GitHub `main` 푸시
6. Vercel 운영 재배포
7. 대표 도메인에서 실제 동작 검증
8. 프로젝트 상태와 작업 기록 갱신

## 구성

### `AGENTS.md`

모든 대화에 자동 적용되는 진입점이다. 다음을 짧고 강하게 규정한다.

- 프로젝트 목적과 사용자 언어
- 세션 시작 시 읽어야 할 상태 파일
- 작업 완료 정의
- 테스트 우선, 사용자 변경 보존, 보안 규칙
- 커밋·푸시·재배포·운영 검증 의무
- 하네스 문서 갱신 조건
- 세부 지침 문서의 위치

### `.agents/PROJECT_STATE.md`

현재 상태의 단일 기준이다.

- GitHub 저장소와 기본 브랜치
- Vercel 팀·프로젝트·대표 도메인
- 현재 운영 배포 상태
- 기술 스택과 핵심 파일
- 완료된 최근 작업
- 미완료 항목과 차단점
- 다음 우선순위

상태가 변하는 작업을 마칠 때 반드시 갱신한다.

### `.agents/ROADMAP.md`

앞으로의 방향성과 우선순위를 기록한다.

- 전환율과 신뢰도를 높이는 랜딩 페이지 개선
- 모바일 사용성
- 최신 매출 증빙 관리
- 결제·문의 동선
- 운영 자동화와 관측성
- 각 단계의 완료 조건과 필요한 자료

### `.agents/RUNBOOK.md`

반복 실행 절차를 명시한다.

- 세션 시작 점검
- UI·콘텐츠 수정 절차
- 매출 자료 교체 절차
- 테스트 명령
- Git 커밋과 푸시
- Vercel 운영 배포
- 대표 도메인에서 데스크톱·모바일 검증
- 장애·롤백 절차
- GitHub 자동 배포 연결 복구 절차

### `.agents/BRAND_RULES.md`

사이트 고유의 시각·콘텐츠 기준을 기록한다.

- 한국어 우선
- 강한 대비와 신뢰 중심의 판매 페이지
- 모바일에서 주요 제목과 CTA의 가독성
- 매출 수치, 기준일, 환율, 이미지의 일치
- 외부 링크와 카카오톡 CTA 기준
- 기존 디자인 언어를 해치는 임의 재설계 금지

### `.agents/LESSONS.md`

이미 겪은 문제와 재발 방지책을 기록한다.

- 잘못된 GitHub 원격 주소
- Vercel GitHub 앱 권한 부족
- 플러그인 배포 도구 입력 스키마 오류
- 미추적 임시 파일의 배포 위험과 `.vercelignore`
- `file://` 브라우저 검증 제한
- PowerShell의 `npm.ps1` 실행 제한과 `npm.cmd`
- 지연 로딩 이미지 검증 시 실제 섹션까지 스크롤해야 함
- Vercel CLI 연결 정보 `.vercel/` 비공개 유지

## 능동 행동 규칙

- 질문에 답하는 것만으로 끝내지 않고 안전한 다음 단계를 실행한다.
- 발견 가능한 정보는 사용자에게 묻지 않고 저장소, Git, Vercel 상태에서 확인한다.
- 사용자가 변경을 요청하면 관련 테스트, 커밋, 푸시, 운영 재배포, 운영 URL 검증까지 한 작업으로 처리한다.
- 결제, 도메인 구매, 권한 승인처럼 사용자의 명시적 승인이 필요한 단계에서만 멈춘다.
- 운영 상태가 바뀌면 `PROJECT_STATE.md`를 갱신한다.
- 새로운 반복 실패나 해결책이 생기면 `LESSONS.md`에 추가한다.
- 방향성이 바뀌면 `ROADMAP.md`를 갱신한다.

## 검증

하네스 작성 후 다음을 확인한다.

- 루트 `AGENTS.md`가 모든 세부 문서를 정확히 연결한다.
- 문서 간 운영 주소, 저장소, 브랜치, 명령이 일치한다.
- 미완성 표식이나 자리표시자가 없다.
- 비밀번호, 토큰, 개인 결제 정보가 없다.
- 현재 저장소에서 실제 실행 가능한 명령만 포함한다.
- 새 대화가 문서만 읽고도 작업의 다음 단계와 완료 조건을 판단할 수 있다.
